## Key Decisions

### i18n Scope: All Feature Composables

**Chosen:** Systematic extraction from all 13 feature screens + core strings

**Rationale:**
- Enables Spanish locale testing in emulator/device
- Lays foundation for future languages (Portuguese, French)
- Composition API's `stringResource()` is composable-friendly and cacheable

### Widget Deferral

**Deferred:** QuickStartWidget, StatsWidget

**Rationale:** 
- Different i18n pattern (Glance uses `context.resources.getString()`)
- Adding widget i18n to scope would block PR completion
- Scoped as separate technical debt task

### String Organization

**Chosen:** Centralized in core module (not scattered across feature modules)

**Rationale:**
- Single source of truth for all string resources
- Easier to manage duplicates and find untranslated strings
- Feature modules import via `com.gymbro.core.R`

## Implementation Notes

**Technical Learnings:**
1. Always add `import androidx.compose.ui.res.stringResource` when extracting strings
2. Always add `import com.gymbro.core.R` for feature module resource references
3. Pluralization (singular "week" vs plural "weeks") requires separate string resources with conditional logic
4. Some screens had mixed hardcoded Spanish already (ProfileScreen, OnboardingScreen) that needed extraction
5. Total hardcoded strings across 15 files: ~100+
